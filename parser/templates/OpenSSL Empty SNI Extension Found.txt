Terdeteksi lalu lintas TLS/SSL yang menggunakan ekstensi SNI (Server Name Indication) kosong pada library OpenSSL. Normalnya, field SNI digunakan untuk menentukan nama host tujuan saat negosiasi TLS, sehingga server dapat memberikan sertifikat yang tepat. Jika nilai SNI kosong, hal ini tidak sesuai praktik standar dan sering dipakai oleh pemindai, bot, atau tool otomatis untuk menyembunyikan tujuan koneksi. Aktivitas ini bisa menunjukkan adanya upaya reconnaissance, koneksi abnormal, atau software yang tidak dikonfigurasi dengan benar.

===============================

Mitigasi :<br>
- Verifikasi apakah ada aplikasi internal yang sengaja menggunakan koneksi tanpa SNI.
- Hardening server TLS/SSL (gunakan versi terbaru, nonaktifkan protokol lama).- Update patch OpenSSL di host terkait.
